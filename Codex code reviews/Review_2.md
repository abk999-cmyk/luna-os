# Review 2

Comprehensive re-audit of the current repository state across the frontend, backend, and `llm-native-os-specs`.

## Critical

- The implemented runtime is still far from the spec: the spec calls for a Rust kernel plus Go middleware/gRPC, WebGPU-backed rendering, and a custom TS/Preact renderer, while the repo is still a single Tauri Rust app with React DOM. See [18-runtime-and-rendering-engine-spec.md](/Users/abhinav/Desktop/OS/llm-native-os-specs/18-runtime-and-rendering-engine-spec.md#L520), [24-technology-stack-and-implementation-guide.md](/Users/abhinav/Desktop/OS/llm-native-os-specs/24-technology-stack-and-implementation-guide.md#L120), [Cargo.toml](/Users/abhinav/Desktop/OS/luna/src-tauri/Cargo.toml#L15), [package.json](/Users/abhinav/Desktop/OS/luna/package.json#L12), [main.tsx](/Users/abhinav/Desktop/OS/luna/src/main.tsx#L1).
- The agent hierarchy is still a stub, not the spec'd system: startup hardcodes one `workspace_default` orchestrator, the leaf is still a stub, and the message protocol lacks the richer envelope the spec requires. See [06-agent-hierarchy-and-orchestration-model.md](/Users/abhinav/Desktop/OS/llm-native-os-specs/06-agent-hierarchy-and-orchestration-model.md#L18), [07-inter-agent-communication-protocol.md](/Users/abhinav/Desktop/OS/llm-native-os-specs/07-inter-agent-communication-protocol.md#L14), [lib.rs](/Users/abhinav/Desktop/OS/luna/src-tauri/src/lib.rs#L195), [leaf.rs](/Users/abhinav/Desktop/OS/luna/src-tauri/src/agent/leaf.rs#L20), [messaging.rs](/Users/abhinav/Desktop/OS/luna/src-tauri/src/agent/messaging.rs#L19).
- The security model is still far below spec, and the current implementation is unsafe by design: there are no selectable security modes, no sandbox subsystem, no workspace isolation, and the permission layer still broadly trusts the conductor. See [20-security-and-sandboxing-model.md](/Users/abhinav/Desktop/OS/llm-native-os-specs/20-security-and-sandboxing-model.md#L23), [permissions.rs](/Users/abhinav/Desktop/OS/luna/src-tauri/src/security/permissions.rs#L17), [permissions.rs](/Users/abhinav/Desktop/OS/luna/src-tauri/src/security/permissions.rs#L76), [security/mod.rs](/Users/abhinav/Desktop/OS/luna/src-tauri/src/security/mod.rs#L1).
- Conductor message-bus handling is still effectively broken: the conductor registers a receiver, but it only logs messages and does not process them, so routed app events and other bus traffic to the conductor are dropped. See [lib.rs](/Users/abhinav/Desktop/OS/luna/src-tauri/src/lib.rs#L181), [lib.rs](/Users/abhinav/Desktop/OS/luna/src-tauri/src/lib.rs#L187), [handler_registry.rs](/Users/abhinav/Desktop/OS/luna/src-tauri/src/action/handler_registry.rs#L400).

## High

- Concurrent requests can still fail with a false "No AI agent configured" because `send_message` and `send_message_streaming` `take()` the conductor out of shared state while work is in flight. See [commands.rs](/Users/abhinav/Desktop/OS/luna/src-tauri/src/commands.rs#L71), [commands.rs](/Users/abhinav/Desktop/OS/luna/src-tauri/src/commands.rs#L138), [commands.rs](/Users/abhinav/Desktop/OS/luna/src-tauri/src/commands.rs#L258), [commands.rs](/Users/abhinav/Desktop/OS/luna/src-tauri/src/commands.rs#L324).
- Action lifecycle accounting is still wrong in two places: actions are marked `Dispatched` before queueing succeeds, and handlerless actions are marked `Completed` by the queue loop. That means history and DB state can claim success for work that never ran. See [dispatcher.rs](/Users/abhinav/Desktop/OS/luna/src-tauri/src/action/dispatcher.rs#L125), [dispatcher.rs](/Users/abhinav/Desktop/OS/luna/src-tauri/src/action/dispatcher.rs#L153), [dispatcher.rs](/Users/abhinav/Desktop/OS/luna/src-tauri/src/action/dispatcher.rs#L199), [lib.rs](/Users/abhinav/Desktop/OS/luna/src-tauri/src/lib.rs#L294).
- The streaming path is still contract-broken: the backend does no final parse/fallback pass after incremental parsing, emits no `stream_id`, and the frontend can open multiple response windows for one stream because `windowId` is set only after `await addWindow(...)`. See [conductor.rs](/Users/abhinav/Desktop/OS/luna/src-tauri/src/agent/conductor.rs#L253), [commands.rs](/Users/abhinav/Desktop/OS/luna/src-tauri/src/commands.rs#L277), [commands.rs](/Users/abhinav/Desktop/OS/luna/src-tauri/src/commands.rs#L318), [App.tsx](/Users/abhinav/Desktop/OS/luna/src/App.tsx#L78), [App.tsx](/Users/abhinav/Desktop/OS/luna/src/App.tsx#L89).
- The window/app event contract is still inconsistent: backend emits `agent-window-close` and `agent-window-focus`, but the frontend does not listen for them, and `agent-window-create` still drops geometry. Destroyed apps can therefore leave stale frontend windows behind. See [handler_registry.rs](/Users/abhinav/Desktop/OS/luna/src-tauri/src/action/handler_registry.rs#L147), [handler_registry.rs](/Users/abhinav/Desktop/OS/luna/src-tauri/src/action/handler_registry.rs#L160), [App.tsx](/Users/abhinav/Desktop/OS/luna/src/App.tsx#L106), [App.tsx](/Users/abhinav/Desktop/OS/luna/src/App.tsx#L177).
- Permanent permission grants are still only reloaded for hardcoded agent IDs, while grants can be persisted for any agent. Those approvals silently disappear after restart for everyone else. See [commands.rs](/Users/abhinav/Desktop/OS/luna/src-tauri/src/commands.rs#L176), [lib.rs](/Users/abhinav/Desktop/OS/luna/src-tauri/src/lib.rs#L79), [lib.rs](/Users/abhinav/Desktop/OS/luna/src-tauri/src/lib.rs#L83).
- Dynamic app input is still trusted too far: `dispatch_app_event` accepts raw frontend identifiers/payloads with no app/component/handler validation, and renderer-side app specs are still assumed valid enough to dereference immediately. One bad app payload can still break the subtree. See [app/commands.rs](/Users/abhinav/Desktop/OS/luna/src-tauri/src/app/commands.rs#L9), [DynamicRenderer.tsx](/Users/abhinav/Desktop/OS/luna/src/renderer/DynamicRenderer.tsx#L15), [DynamicRenderer.tsx](/Users/abhinav/Desktop/OS/luna/src/renderer/DynamicRenderer.tsx#L29), [app/lifecycle.rs](/Users/abhinav/Desktop/OS/luna/src-tauri/src/app/lifecycle.rs#L86).
- Dynamic app persistence is still missing even though the DB schema creates related tables. `AppManager` remains in-memory only, so running apps disappear on restart. See [db.rs](/Users/abhinav/Desktop/OS/luna/src-tauri/src/persistence/db.rs#L120), [db.rs](/Users/abhinav/Desktop/OS/luna/src-tauri/src/persistence/db.rs#L131), [app/lifecycle.rs](/Users/abhinav/Desktop/OS/luna/src-tauri/src/app/lifecycle.rs#L18).
- The content-type contract is still inconsistent: the frontend can request types like `dynamic_app`, `scratchpad`, and `terminal`, but backend `create_window` still only maps four values and silently coerces the rest to `Empty`. See [windowStore.ts](/Users/abhinav/Desktop/OS/luna/src/stores/windowStore.ts#L48), [window/commands.rs](/Users/abhinav/Desktop/OS/luna/src-tauri/src/window/commands.rs#L26).

## Medium

- Voice and ingestion paths still have real correctness/safety issues: voice stop can miss the final speech-recognition result, `transcribe_audio` has no size/format guard, and dropped unknown files are fully read as text. See [useVoiceInput.ts](/Users/abhinav/Desktop/OS/luna/src/hooks/useVoiceInput.ts#L89), [useVoiceInput.ts](/Users/abhinav/Desktop/OS/luna/src/hooks/useVoiceInput.ts#L118), [commands.rs](/Users/abhinav/Desktop/OS/luna/src-tauri/src/commands.rs#L336), [contextParser.ts](/Users/abhinav/Desktop/OS/luna/src/services/contextParser.ts#L16), [contextParser.ts](/Users/abhinav/Desktop/OS/luna/src/services/contextParser.ts#L30).
- Sync robustness is still weak: `initSyncManager()` can wedge permanently after one listener-registration failure, and one throwing subscriber can abort delivery for later subscribers in the same batch. See [SyncManager.ts](/Users/abhinav/Desktop/OS/luna/src/sync/SyncManager.ts#L15), [SyncManager.ts](/Users/abhinav/Desktop/OS/luna/src/sync/SyncManager.ts#L54).
- Window state still drifts from backend truth: the store ignores authoritative IPC return values for move/resize/restore, maximize is frontend-only, restored windows do not recompute `focusedWindowId`, and off-screen positions are persisted as-is. See [windowStore.ts](/Users/abhinav/Desktop/OS/luna/src/stores/windowStore.ts#L43), [windowStore.ts](/Users/abhinav/Desktop/OS/luna/src/stores/windowStore.ts#L101), [Window.tsx](/Users/abhinav/Desktop/OS/luna/src/components/Window.tsx#L92), [db.rs](/Users/abhinav/Desktop/OS/luna/src-tauri/src/persistence/db.rs#L216).
- The permission UI is still fragile: it dequeues requests even if approve/deny IPC fails, lacks dialog semantics/focus management, and window controls are still unlabeled icon-only buttons. See [PermissionDialog.tsx](/Users/abhinav/Desktop/OS/luna/src/components/PermissionDialog.tsx#L18), [PermissionDialog.tsx](/Users/abhinav/Desktop/OS/luna/src/components/PermissionDialog.tsx#L45), [Window.tsx](/Users/abhinav/Desktop/OS/luna/src/components/Window.tsx#L146).
- There are still UX/styling regressions: global `user-select: none` blocks copying normal UI text, and `sprint2.css` still references undefined spacing/error/font tokens. See [reset.css](/Users/abhinav/Desktop/OS/luna/src/styles/reset.css#L16), [sprint2.css](/Users/abhinav/Desktop/OS/luna/src/styles/sprint2.css#L10).
- Test coverage is still far below what the spec requires: the frontend has no test script, and Rust still only has the two topic-matching tests. See [25-testing-strategy-and-quality-assurance.md](/Users/abhinav/Desktop/OS/llm-native-os-specs/25-testing-strategy-and-quality-assurance.md#L11), [package.json](/Users/abhinav/Desktop/OS/luna/package.json#L6), [topic.rs](/Users/abhinav/Desktop/OS/luna/src-tauri/src/sync/topic.rs#L66).
